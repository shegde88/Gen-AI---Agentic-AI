# FIFA World Cup 2026 Fan Intelligence RAG App

**Gen Academy Mastering Agentic AI — Week 2 Project**
**Build track:** Code-heavy (LangChain + LangGraph + Pinecone)

> My RAG app helps football fans answer questions about teams, players, venues, match history, and training camp locations from 96 curated Wikipedia sources + 7 structured CSV datasets in a Streamlit chat UI with 15/15 questions passing and 83% keyword coverage.

---

## Architecture

```
User question
     │
     ▼
Query pre-processing
  └── Date expansion: "June 11th" → "June 11th (2026-06-11)"
  └── Follow-up detection: short contextual questions bypass retrieval gate
     │
     ▼
Dense retriever (Pinecone cosine, k=20 candidates)
     │
     ▼
Cohere rerank-english-v3.0 (cross-encoder → top-5)
     │
     ▼
LangGraph RAG graph — 4 nodes
  ├── retrieve node           (fetch + deduplicate + filter footnote chunks)
  ├── grade_relevance         (Cohere score < 0.25 → refuse; follow-up bypass)
  ├── answer node        ───► Nebius Llama-3.3-70B-Instruct
  ├── check_hallucination     (LLM verifies answer is grounded; retry up to 2×)
  └── refuse node        ───► "I couldn't find this in the World Cup knowledge base."
     │
     ▼
Cited answer (top-3 sources scoring ≥ 0.40)
```

---

## Tech Stack

| Layer | Choice |
|---|---|
| Embeddings | Nebius `Qwen/Qwen3-Embedding-8B` (4096d) |
| Vector store | Pinecone serverless (AWS us-east-1, cosine) |
| Reranker | Cohere `rerank-english-v3.0` cross-encoder |
| LLM | Nebius `meta-llama/Llama-3.3-70B-Instruct` |
| Framework | LangChain + LangGraph |
| UI | Streamlit (primary) + HTML/JS frontend (bonus) |

## Setup

### 1. Install dependencies

```bash
pip install uv
uv venv && source .venv/bin/activate
uv pip install -e .
```

### 2. Configure API keys

```bash
cp .env.example .env
# Edit .env and fill in: NEBIUS_API_KEY, PINECONE_API_KEY, COHERE_API_KEY
```

### 3. Build the Pinecone index (one-time)

```bash
python scripts/ingest.py
```

This will:
- **Clear** the existing Pinecone index (clean re-ingest, no stale duplicates)
- Fetch 96 Wikipedia articles (~8–10 minutes with polite crawl delays)
- Load 7 CSV datasets from `data/raw/`
- Chunk into ~512-token passages with 50-token overlap
- Embed with `Qwen/Qwen3-Embedding-8B` via Nebius
- Upsert to a Pinecone index named `fifa-wc-2026`

### 4. Launch the app

```bash
streamlit run app.py
```

### 5. (Optional) Bonus HTML/JS frontend

```bash
uvicorn api:app --reload --port 8000
# Then open frontend/index.html in your browser
```

## Corpus Sources

### Wikipedia (96 articles)

| Category | Count | Details |
|---|---|---|
| Tournament | 15 | 2026 overview (with training camps + venues), Groups A–L (match schedules, venues per group), squads, venues |
| History | 24 | FIFA WC overview, records + all 22 editions (1930–2022) |

| File | Content |

Each row is enriched into a natural language summary before indexing so location-based queries ("which team is based in Washington state?") score correctly against individual team chunks rather than dense table excerpts.

## Key RAG Design Decisions

| Decision | Rationale |
|---|---|
| LangGraph 4-node StateGraph | Enables conditional routing, hallucination retry loops, and a first-class refusal path |
| `_expand_date_in_query()` | Bridges "June 11th" (natural language) ↔ "2026-06-11" (ISO dates in CSV) at query time — no re-ingest needed |
| Follow-up bypass in `grade_relevance` | Short contextual questions ("Yes in PST?") route to the answer node via `chat_history` instead of being refused for having no Pinecone match |
| `CITATION_TOP_N=3` + `CITATION_MIN_SCORE=0.40` | Only top-3 docs scoring ≥ 0.40 are shown as sources — prevents marginal historical chunks (1994/2006 WC) from appearing as citations |
| `training_camps_2026.csv` with per-team enrichment | Dense Wikipedia table chunks score poorly for location queries; one dedicated chunk per team fixes this |
| `clear_index()` before every re-ingest | Prevents stale or duplicate vectors accumulating across ingest runs |
